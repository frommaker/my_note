1.若没有文件，先打开vscode，在设置-命令面板上查找git clone命令

2.输入ssh链接，确定，选择存储路径即可
此时克隆的是默认的main分支，如果想要克隆别的分支，需要在vscode终端上cd F:\Git_Project
然后输入git clone -b <分支名> <Git 仓库 URL> ，此时会将该分支克隆到前面的文件夹处，名字为项目名
打开已有文件后会询问是否打开父存储库，选择打开

3.若本身之前已经有文件，先打开，然后点击源代码管理，点击三个点然后点击拉取，即可拉取最新云端内容
（每次开始工作前都拉取一次）

4.修改代码并保存后可以在源代码管理那里的更改处选择更改项，然后在上面输入说明信息并点击提交
（原本还需要点击更改项右边的加号进行暂存，但如果尝试一次不暂存后，并点击始终这样，就不用暂存了）

5.然后点击三个点--推送 即可（或拉取与推送--推送到--仓库名   或 终端输入git push origin 分支名）
（需要等一到十几分钟才能在云端看到）

6.注意左下角为分支名

7.回退版本，先在git log --oneline 查看版本更迭信息，然后在vscode终端输入git reset --hard HEAD^n 来回退n次，回退后无法查看未来版本的更迭信息，或用git reset --hard 版本号 来直接跳回某个版本（版本号输入前几个数即可）。
若没记住之前的回退版本操作，可以输入git reflog来查看这个分支对版本的操作记录

8.若部分内容尚未完成，且需要回到另一个版本，可以先对更改内容存储，点击三个点选择存储-储存（包含未跟踪），可选择填写文件名或不填，在另一个版本结束操作后跳回该版本，点击三个点选择存储-弹出存储，选择已有项或输入文件名即可弹出。（注意提交到暂存的文件不会在切换版本时保存，必须stash，如果commit，则会新增一个版本）

9.想要撤销工作区的修改，使用命令git checkout -- 文件名 （注意--前后空格），若该文件之前没放到暂存区，则回溯到仓库的状态，若之前放到暂存区又做了修改，则回溯到暂存区的状态。（即回溯到最后一次commit或add的状态）
如果想要将暂存区的状态回溯到仓库，可以git reset HEAD 文件名，（HEAD表示最新版本），此时工作区的文件不会变，原先暂存的文件会变成M，即修改的文件。
若删除了一些文件，同样可以用git checkout -- 文件名来还原。
（最新版的git使用git switch和git restore来分别进行分支创建切换和文件恢复操作）

10.如果删除了一些文件（手动或者git rm 文件名）,可以在源代码管理那里commit，然后提交，
也可以在终端，但必须先git add ./添加到暂存区，才能commit，不然commit然后push上去的是原来的代码。

11.可以用git tag 标签名 来对当前项目起标签，如v1.0，标签仅针对commit版本，即另一个分支也可能可以看到；
也可以用git tag v0.9 f52c633 来创建标签，或者加上说明信息：git tag -a v0.1 -m "version 0.1 released" 1094adb
可以用git tag 来显示所有标签，用git show 标签名 来显示该标签的所有信息。
创建完标签后，通过git push origin 标签名 来推送到云端

12.将别的项目弄到自建仓库：
1.通过git clone git@github.com:frommaker/my_odrive.git克隆自建仓库到本地
2.删除fw-v0.5.1版本的.git文件夹，然后复制剩下的到该文件夹
3.通过add,commit,push将其推送到云端
4.通过git checkout 分支 新增my_branch分支，并同上一步将其上传到云端
5.接下来通过vscode来修改内容

13.在vscode打开一个分支后，若创建新分支，此时新分支的内容与旧分支相同，直接push即可上传到云端；
若在一个分支上做了修改，需要储存或者提交后才能正常切换到另一个分支，强制切换会还原之前的修改。
在一个分支上创建了文件，提交后，切换到另一个分支，这个文件不会在文件夹中出现，相当于删去了。

14.合并分支就是将两个分支的内容加和起来，有冲突的地方要修改掉，合并后两个分支的内容和提交历史一样

15.想要将单独一个文件push上去，可以git add 文件名，如果该文件被gitgnore文件忽略，可以加个-f前缀再add

16.某个分支可以通过git push origin 分支名 来拉取别的分支或者自己的云端项目，如果有冲突会放在暂存窗口，左边是本地的，右边是拉取的云端的，不操作直接commit会以云端为主，要更改则先将该文件取消暂存，更改后再commit。
